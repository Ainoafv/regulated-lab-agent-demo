Governed AI agent — QC batch workflow

An AI agent handles routine QC requests on batch B-2047 — result summaries, LIMS look-ups, draft ELN entries — under a permission policy enforced in code. The policy, not the model, decides what executes automatically, what is held for human sign-off, and what is never automated (batch release, LIMS status changes). Fictional scenario — not Thermo Fisher software.

Who it serves — and who stays in control

The same workflow has to satisfy three stakeholders at once. The agent gives each what they need — without ever crossing the line the others care about.

ScientistSpeed. Instant summaries and answers, less manual paperwork — self-serve, read-only actions, never blocked by a queue.
QA & ComplianceControl. No AI touching the quality record or batch disposition unsupervised — human sign-off on every write, full audit trail.
ITSafe integration. A closed action set and a policy in code — no open-ended writes, fail-closed by default.
Overview · 3 / 3

Why I built this

I come from science and bioinformatics, and I now build and integrate AI systems end-to-end. This prototype is how I think about the future of the connected lab: automating the repetitive work (summaries, look-ups, draft notes), integrating across LIMS / ELN / instruments, and keeping a human in control of anything that touches the quality record — the same direction the industry is taking with QC-analyst agents, lab orchestration and responsible-AI governance.

My edge isn't another LIMS CV — it's seeing the workflow from the end user's side (scientist, QA, IT) and being able to design, build and explain it to each of them.
